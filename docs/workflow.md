# 数据流设计

## 食材储藏影响饭菜计划

1. 读取当前可用库存。
2. 找出即将过期的食材。
3. 含有临期食材的菜谱提高优先级。
4. 库存覆盖率高的菜谱提高优先级。
5. 命中忌口或不喜欢食材的菜谱降低或排除。
6. 根据家庭成员年龄、性别和份量系数计算每餐需要准备的总份量。
7. 晚餐如果命中库存中的整份肉类或鱼虾，会尽量把计划份量调整到用完整份，减少半袋半份残留。

## 饭菜计划生成采购计划

1. 读取一周 14 餐计划。
2. 汇总每道菜所需食材。
3. 减去当前可用库存。
4. 缺口写入采购清单。
5. 主食材缺失标记为高优先级。

## 总结反馈更新库存和未来推荐

1. 每餐记录实际是否完成、评分和备注。
2. 按实际消耗写入食材消耗日志。
3. 扣减库存数量。
4. 库存为 0 时标记为 `used_up`。
5. 汇总菜谱评分，影响未来排序。

## MVP 闭环

```text
录入库存
-> 生成一周饭菜计划
-> 生成采购清单
-> 记录实际做饭和评分
-> 扣减库存
-> 生成一周总结
-> 下周计划参考历史反馈
```
